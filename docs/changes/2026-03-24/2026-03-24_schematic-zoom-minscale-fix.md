## 变更前问题

原理图预览的缩放逻辑使用固定的最小缩放值 `0.25`，当页面真实整页适配比例低于该值时，用户一旦进行缩放就会被强行跳到 `25%`，且无法通过 `Zoom Out` 回到真实整页视图。

## 变更内容

调整原理图预览的最小缩放约束，改为基于当前页面的 `fit page` 比例动态计算；同时将状态栏缩放显示改为保留一位小数，避免低比例被误显示为 `0%`。

## 影响范围

影响 GUI 原理图预览的缩放下界和状态栏缩放显示，不改变页面内容、拖动交互和渲染数据结构。

## 验证结果

执行 `npm run build`（目录：`src/gui`），构建通过。

## 下一步建议

建议后续将最小缩放、最大缩放和工具栏步进统一提取为 viewport 常量，避免交互参数继续散落在组件逻辑中。
