## 变更前问题

原理图预览从整页模式执行首次 `Zoom In` 时，虽然已经对准内容区域，但仍可能直接采用内容适配比例作为缩放基数，导致页面一下跳到器件几乎铺满屏幕的状态。

## 变更内容

调整原理图预览的首次放大逻辑：整页模式下保持围绕内容中心放大，但缩放基数改为当前视口比例，仅进行渐进式放大，不再直接跳到内容适配比例。

## 影响范围

影响 GUI 原理图预览从整页模式进入放大模式时的缩放步进体验，不改变页面内容、拖动约束和基础渲染逻辑。

## 验证结果

执行 `npm run build`（目录：`src/gui`），构建通过。

## 下一步建议

如果后续还需要更细的控制，建议把工具栏缩放步长暴露成常量，并区分“首次放大步长”和“连续放大步长”。
