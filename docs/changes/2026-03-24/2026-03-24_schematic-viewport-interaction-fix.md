## 变更前问题

原理图预览在拖动和放大时缺少视口约束，缩放也默认围绕左上角进行，导致用户一旦交互就容易把页面甩出可视区，出现“无法正常显示”的现象。

## 变更内容

调整原理图预览的缩放逻辑，使其围绕视口中心进行；同时为拖动和缩放后的平移增加边界约束，保证页面在交互后仍然保留可见区域。

## 影响范围

影响 GUI 原理图预览的拖动与缩放交互体验，不改变原理图几何数据、页面布局和符号渲染内容。

## 验证结果

执行 `npm run build`（目录：`src/gui`），构建通过。

## 下一步建议

如果后续要继续增强交互，建议补充鼠标滚轮缩放和双击定位，并为 viewport 约束逻辑添加单元测试。
