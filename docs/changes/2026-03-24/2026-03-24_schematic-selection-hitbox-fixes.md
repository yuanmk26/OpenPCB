# 原理图选中命中修复

## 变更前问题
- 点击网络名称时，右侧栏无法稳定显示网络信息，标签也没有正常高亮。
- 器件的可点击区域过小，`RefDes`、`Value` 等可见区域点击后不会选中器件。

## 变更内容
- 将网络标签的点击和阻止冒泡逻辑提升到标签分组层，保证点击文本本体也能触发选中。
- 保留网络标签透明命中框，并让选中高亮跟随当前标签状态显示。
- 将器件点击和阻止冒泡逻辑提升到器件分组层。
- 扩大器件透明命中区域，使其覆盖符号主体以及 `RefDes`、`Value` 文本显示区域。

## 影响范围
- `src/gui/ui/components/placeholders/SchematicPreview.tsx`

## 验证结果
- 已执行前端构建验证，确认 TypeScript 类型检查和 Vite 打包通过。
- 已检查交互逻辑，覆盖网络标签点击选中、器件文本区域点击选中和空白区域取消选中场景。

## 下一步建议
- 后续可继续为引脚名称和导线段补充一致的命中策略，避免不同元素的交互热区体验不统一。
