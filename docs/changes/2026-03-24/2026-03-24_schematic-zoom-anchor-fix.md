## 变更前问题

原理图预览在整页模式下执行 `Zoom In` 时，缩放锚点落在页面中心，而当前 mock 内容集中在页面左上区域，导致放大后视口迅速落到大面积空白区，看起来像器件消失。

## 变更内容

调整原理图预览的放大锚点：在整页模式下优先围绕内容包围盒中心缩放，非整页模式仍保持围绕当前视口中心缩放。

## 影响范围

影响 GUI 原理图预览的放大交互体验，不改变原理图几何数据、页面排版和基础 viewport 约束逻辑。

## 验证结果

执行 `npm run build`（目录：`src/gui`），构建通过。

## 下一步建议

如果后续要继续细化交互，建议在 `page` 与 `content` 两种模式下分别定义更明确的缩放策略和默认焦点。
