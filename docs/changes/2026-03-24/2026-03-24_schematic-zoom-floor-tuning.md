## 变更前问题

原理图预览将最小缩放额外限制在固定的 `1%`，当页面真实整页适配比例低于该值时，会导致用户无法回到合适的全貌视图，器件在低缩放下仍显得过大。

## 变更内容

移除固定的 `1%` 缩放下限，改为基于当前页面 `fit page` 比例动态计算最小缩放，并允许略微低于整页适配比例；同时优化状态栏缩放显示格式，使低比例缩放能显示更精确的百分比。

## 影响范围

影响 GUI 原理图预览的最小缩放行为和状态栏缩放展示，不改变页面内容、交互模式和渲染数据结构。

## 验证结果

执行 `npm run build`（目录：`src/gui`），构建通过。

## 下一步建议

建议后续将缩放下限、上限和步长整理到统一的 viewport 配置常量中，并按页面类型做差异化调优。
