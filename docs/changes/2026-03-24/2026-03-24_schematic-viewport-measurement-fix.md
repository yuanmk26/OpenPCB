## 变更前问题

原理图预览在部分布局情况下会错误地拿到接近 0 或 1 的容器尺寸，导致 viewport fit 退化为 100% 缩放和异常平移，表现为只能看到页面中间局部、定位圆消失、调试坐标只剩 `0 0 origin`。

## 变更内容

改用 `getBoundingClientRect()` 作为原理图视口尺寸测量兜底；在 `ResizeObserver` 返回异常小尺寸时回退到真实布局尺寸；同时为原理图舞台增加明确高度，并收紧 fit 计算，避免可用宽高为负时回退到 `scale = 1`。

## 影响范围

影响 GUI 原理图预览的容器测量、初始 fit 和窗口内显示稳定性，不改变原理图几何数据本身。

## 验证结果

执行 `npm run build`（目录：`src/gui`），构建通过。

## 下一步建议

如果后续还出现极端缩放问题，建议在状态栏临时输出实际 `containerSize`，进一步确认布局树中的尺寸传递是否稳定。
