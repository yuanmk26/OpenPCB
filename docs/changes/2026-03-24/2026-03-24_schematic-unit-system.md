## 变更前问题

原理图预览中的页面尺寸、符号尺寸、网格步距、文字字号、线宽和 viewport padding 分散在多个文件中，且大量依赖经验值，导致比例问题难以判断和修正。

## 变更内容

新增统一的原理图度量常量文件，定义 scene unit、网格步距、页框尺寸、线宽、半径和文本字号；将 viewport 适配、场景构建、SVG 渲染参数和 mock 原理图几何全部切换到这套单位体系。

## 影响范围

影响 GUI 原理图预览的几何定义方式、渲染度量和 mock 数据表达方式，不影响项目其它视图和 Tauri schema 文件。

## 验证结果

执行 `npm run build`（目录：`src/gui`），构建通过。

## 下一步建议

后续接入真实原理图数据时，建议明确后端输出单位与当前 scene unit 的映射关系，并补充 viewport 与几何比例的单元测试。
