## 变更前问题

原理图预览在整页模式下执行 `Zoom In` 时，仍可能沿用整页视角作为缩放基础，导致放大后优先看到页面框线而不是器件内容，表现为器件消失、拖动时只看到框框在移动。

## 变更内容

调整原理图预览的放大过渡逻辑：当用户从整页模式开始放大时，先以内容适配视角作为缩放基础，再执行放大和视口约束。

## 影响范围

影响 GUI 原理图预览从整页视角进入放大视角时的过渡行为，不改变原理图几何数据、页面内容和拖动约束逻辑。

## 验证结果

执行 `npm run build`（目录：`src/gui`），构建通过。

## 下一步建议

建议后续把工具栏缩放、滚轮缩放和双击缩放统一为同一套从 `page` 过渡到 `content/manual` 的策略。
