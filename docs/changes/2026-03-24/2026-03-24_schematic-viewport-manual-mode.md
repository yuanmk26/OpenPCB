## 变更前问题

原理图预览在用户执行缩放或拖动后，仍可能因为容器尺寸变化或自动 fit 逻辑重新计算视口，导致视图一会儿正常、一会儿突然跳回异常位置；同时按钮缩放步长偏大，容易放大过头。

## 变更内容

为原理图预览新增 `manual` 视口模式，在用户手动缩放或拖动后停止自动 fit，仅保留边界约束；页面切换时再恢复整页适配。同时将按钮缩放步长从 `1.2` 调整为 `1.1`。

## 影响范围

影响 GUI 原理图预览的缩放与拖动交互稳定性，不改变原理图几何数据、页面内容和基础渲染规则。

## 验证结果

执行 `npm run build`（目录：`src/gui`），构建通过。

## 下一步建议

建议后续把按钮缩放、滚轮缩放和双击缩放统一到同一套 viewport 策略，并补充对应的交互测试。
