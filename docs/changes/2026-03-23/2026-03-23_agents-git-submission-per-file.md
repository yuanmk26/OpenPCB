# AGENTS Git 提交流程按文件强制化

## 变更前问题
- 现有 Git 规则仅说明“有文件变更时默认执行 add/commit/push”，未明确到“每一个新增或修改文件”都必须走提交流程。
- 在多文件任务中，规则颗粒度不够严格，容易产生理解偏差。

## 变更内容
- 更新 `AGENTS.md` 的 `Git Submission Rules`：
  - 明确每一个新增或修改文件都必须执行完整 Git 提交流程。
  - 明确每个文件的必需流程为：`git add <file>` + `git commit` + `git push`。
  - 明确多文件任务中不得跳过任一变更文件的提交流程。
  - 保留 push 失败需保留错误输出并说明原因、用户显式要求不提交可例外两条规则。

## 影响范围
- 影响仓库全局代理协作与提交流程约束。
- 不影响业务代码逻辑与运行行为。

## 验证结果
- 已检查 `AGENTS.md` 内容，`Git Submission Rules` 已按“按文件强制提交流程”更新。
- 本次变更已补充对应 `docs/changes/2026-03-23/` 记录，满足维护规则。

## 下一步建议
- 在后续任务中按该规则执行时，可统一约定 commit message 模板，提升提交历史可读性。
