# 2026-03-10 维护规则新增自动提交推送

## 变更前问题
- 现有文档维护规则未明确要求任务完成后自动执行 Git 提交与推送。
- 协作流程中“改完是否已提交到远端”缺少统一约束。

## 变更内容
- 在 `docs/README.md` 的维护规则中新增：
  - 每次任务完成后，默认执行 `git add` + `git commit` + `git push`。
  - 推送失败时需明确报错原因。
- 在 `docs/architecture/README.md` 的维护规则中同步新增同样约束，保持规则一致。

## 影响范围
- 仅文档流程规则变更，不影响代码运行逻辑与 CLI 行为。
- 后续每次修改任务默认进入“完成后提交并推送”的收尾流程。

## 验证结果
